# task-003 成果物確認UIとFileRef最小表示

作成日: 2026-05-31

## 目的

- task-001のdry-runで作られた成果物参照を、人間がUIで確認できるようにする。
- 最初は成果物本体ではなく、工程、要約、参照URI、状態だけを表示する。
- UIが依頼と状態だけで終わらず、最後まで実行された結果を確認できるようにする。

## 作業スコープ

今回やること:

- UIに成果物一覧または最新成果物の詳細表示を追加する。
- 成果物種別、参照URI、作成元工程、短い意味を表示する。
- dry-run成果物であることが分かる表示にする。
- 巨大なSTT全文、LLM全文、動画本体をUIへ埋め込まない。
- 成果物がない場合と全工程完了後の表示を確認する。

今回やらないこと:

- 成果物本体のビューアは作らない。
- 動画プレビューは作らない。
- STT全文、LLM全文、Gemini評価全文の表示は作らない。
- 投稿、採用、自動判定は作らない。

## 完了条件

- dry-run完了後に成果物参照がUIで確認できる。
- どの工程の成果物か分かる。
- 成果物本体を状態やUIへ巨大に埋め込んでいない。
- `pnpm run type-check` と画面確認が成功する。
