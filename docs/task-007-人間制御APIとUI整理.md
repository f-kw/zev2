# task-007 人間制御APIとUI整理

作成日: 2026-05-31
更新日: 2026-06-29

## 目的

- 人間がAIエージェントの実行を止める、差し戻す、再実行する、却下するためのAPIを用意する。
- UIはAIエージェント実行の主導権を持たず、人間制御と状態確認に絞る。
- 「人間が承認し、AIエージェントがAPIで実行する」という役割を画面とAPIで崩さない。

## 作業スコープ

今回やること:

- 依頼の却下APIを追加する。
- 作業のキャンセルAPIを追加する。
- 失敗作業またはキャンセル作業の再実行APIを追加する。
- 人間の差し戻し理由を保存できるようにする。
- UIには、承認、却下、停止、再実行、状態確認だけを出す。
- UIにAIエージェント作業を1件ずつ進める実行ボタンを戻さない。

今回やらないこと:

- AIエージェントrunnerの実処理を実装しない。
- STT、LLM、動画生成を実行しない。
- 詳細ログビューアや成果物ビューアはこのタスクでは作らない。
- 認証と権限は別タスクで扱う。

## 完了条件

- 人間がAPIで依頼を却下できる。
- 人間がAPIで実行中または待機中の作業をキャンセルできる。
- 人間がAPIで失敗作業を再実行待ちへ戻せる。
- UIは依頼、承認、状態確認、人間制御だけに絞られている。
- UI操作なしでもAIエージェントはAPIで作業を進められる。
- `pnpm run type-check` と対象テストが成功する。

## 実装結果

- `POST /api/request-drafts/:id/reject` で、未承認の実行前下書きを理由つきで却下できる。
- 却下済み下書きは承認できない。却下時にAI作業キューは作らない。
- 却下理由はAI操作ログと作業履歴APIから確認できる。
- `POST /api/request-drafts/:id/cancel-agent-work` で、実行中または待機中のAI作業を中止できる。
- 人間確認待ちの状態で中止した場合は、開いている確認項目を中止による却下として閉じる。
- `POST /api/agent-requests/:id/retry` で、失敗したAI工程から新しい編集コピーを作り、失敗工程以降を再実行できる。
- 確認画面の承認、却下、作り直し理由は人間判断として保存する。
- UIは新規依頼を作ると自動で動画作成へ進む主導線にしているため、実行前下書きの却下ボタンは主画面へ増やしていない。
- UIには、内容確認、使用素材確認、動画生成前確認、生成後の作り直し、AI作業中止、状態確認を置く。
- UIからAI工程を1件ずつ進める実行ボタンは戻していない。

## 確認方法

- `pnpm run type-check`
- `pnpm run scenario:agent`
- `pnpm test`

シナリオテストでは、理由なし却下の拒否、下書き却下後にAI作業が作られないこと、却下済み下書きを承認できないこと、人間確認の却下、中止、失敗工程の再実行コピーを確認する。
