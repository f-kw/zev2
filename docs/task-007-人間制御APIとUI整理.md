# task-007 人間制御APIとUI整理

作成日: 2026-05-31

## 目的

- 人間がAIエージェントの実行を止める、差し戻す、再実行する、却下するためのAPIを用意する。
- UIはAIエージェント実行の主導権を持たず、人間制御と状態確認に絞る。
- 「人間が承認し、AIエージェントがAPIで実行する」という役割を画面とAPIで崩さない。

## 作業スコープ

今回やること:

- 依頼の却下APIを追加する。
- 作業のキャンセルAPIを追加する。
- 失敗作業またはキャンセル作業の再実行APIを追加する。
- 人間の差し戻し理由を保存できるようにする。
- UIには、承認、却下、停止、再実行、状態確認だけを出す。
- UIにAIエージェント作業を1件ずつ進める実行ボタンを戻さない。

今回やらないこと:

- AIエージェントrunnerの実処理を実装しない。
- STT、LLM、動画生成を実行しない。
- 詳細ログビューアや成果物ビューアはこのタスクでは作らない。
- 認証と権限は別タスクで扱う。

## 完了条件

- 人間がAPIで依頼を却下できる。
- 人間がAPIで実行中または待機中の作業をキャンセルできる。
- 人間がAPIで失敗作業を再実行待ちへ戻せる。
- UIは依頼、承認、状態確認、人間制御だけに絞られている。
- UI操作なしでもAIエージェントはAPIで作業を進められる。
- `pnpm run type-check` と対象テストが成功する。
