# task-008 Gemini APIで演出作成

## 目的

選ばれたテーマに関係する複数の動画箇所をGemini APIへ渡し、つなぎ方、テロップ、間、強調、カット上の注意点を含む演出案を作れるようにする。

候補決定はこのタスクの目的ではない。
候補決定は、文字起こしからテーマ候補を出し、人間が切り抜きたいテーマを選ぶ流れで行う。

## 前提

- 文字起こしからテーマ候補を出す。
- 人間が切り抜きたいテーマを選ぶ。
- 選ばれたテーマに関係する複数の発話箇所を集める。
- 複数箇所をどうつなぐかの構成案を作る。
- 演出作成では、その構成案に含まれる複数の動画箇所をGemini APIへ渡す。

## やること

- Gemini APIへ渡す複数動画箇所の入力形式を決める。
- Gemini APIから受け取る演出案の保存形式を決める。
- テロップ、つなぎ、間、強調、カット上の注意点を構造化して保存する。
- Gemini APIの出力だけで自動採用せず、人間が動画生成前確認で進める、直す、止めるを選べるようにする。

## やらないこと

- Web版Geminiで演出作成しない。
- 完成品レビューをこのタスクに含めない。
- 音声、テンション、間、笑い、画面変化、映像の見栄えを候補決定の主材料にしない。
- 高コストな補助計測を前段に追加しない。
- 独自の係数、重み、総合点で自動判断しない。

## 完了条件

- 演出作成の入力が、単一箇所ではなく複数動画箇所を扱える。
- Gemini APIへ渡す内容が、選ばれたテーマと構成案に紐づいている。
- Gemini APIの出力が、動画生成前確認で人間が読める演出案として表示できる。
- backendがGemini APIを直接実行しない境界が保たれている。
