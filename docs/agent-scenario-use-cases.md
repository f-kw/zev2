# AIエージェントシナリオテストのユースケース

作成日: 2026-06-22

## 目的

zev2 の現段階の目的は完成UIではなく、人間承認とAIエージェント実行骨格を確認すること。
このテストでは、外部サービスの品質ではなく、人間確認工程、型付きAI工程、成果物参照、失敗時の扱い、作り直し時の状態遷移を確認する。

## P0 人間承認とAI工程骨格

### UC-P0-01 依頼下書きは承認されるまでAI工程を開始しない

- 前提: 依頼タブで対象動画、目的、尺、候補数、Geminiモデル、プリセットを入力する。
- 操作: 下書きを作成し、承認前後のAI工程を確認する。
- 期待結果:
  - 承認前はAI工程が作られない。
  - 承認後は動画生成以外のAI工程がキューに入り、動画生成は作られない。
  - AI工程には依頼内容と設定が渡る。
  - 同じ下書きの二重承認は拒否される。

### UC-P0-02 サンプル動画の正常系が動画生成前確認まで到達する

- 前提: 対象動画が `zev-sample://speech-id-timing` である。
- 操作: 下書き承認、内容選択、使用素材構成、使用素材確認、演出作成、微調整まで進める。
- 期待結果:
  - 内容候補成果物の作成方法は `sample-theme-options` になる。
  - 演出案成果物の作成方法は `sample-edit-plan` になる。
  - 構成案、演出案、微調整成果物がファイル参照として保存される。
  - 動画生成前確認が確認待ちになる。

### UC-P0-03 内容選択までは使用素材構成に進まない

- 前提: STT後、内容候補の確認が表示されている。
- 操作: 内容選択前後の次AI工程を確認する。
- 期待結果:
  - 内容選択前は使用素材構成が実行されない。
  - 内容選択後、使用素材構成が次のAI工程になる。
  - 構成案には選んだ内容候補IDが反映される。

### UC-P0-04 実動画の内容候補整理は外部AIなしで成立する

- 前提: 対象動画が `zev-sample://` ではなく、文字起こし成果物がある。
- 操作: 文字起こし済みの状態から内容候補整理を実行する。
- 期待結果:
  - Gemini API接続情報なしでも、文字起こしから内容候補を整理できる。
  - 内容候補は面白さを断定せず、何があるかの説明として保存される。
  - 内容選択確認が作られる。
  - 内容選択前は使用素材構成へ進まない。

### UC-P0-05 実動画の演出作成はGemini APIまたは動画断片なしで失敗する

- 前提: 実動画相当の構成案までできているが、動画断片またはGemini API接続情報がない。
- 操作: 演出作成を実行する。
- 期待結果:
  - 演出作成は失敗する。
  - `sample-edit-plan` に逃げない。
  - 演出案成果物を保存しない。
  - 微調整と動画生成前確認へ進まない。

### UC-P0-06 動画生成前確認の承認で動画生成へ進む

- 前提: 動画生成前確認が確認待ちである。
- 操作: 「この編集案で作る」に相当する承認を送る。
- 期待結果:
  - 承認前は動画生成AI工程が存在しない。
  - 承認後に動画生成AI工程が作られる。
  - 承認済みの生成前確認を二重承認できない。

### UC-P0-07 生成前確認から使用素材構成と演出を作り直せる

- 前提: 動画生成前確認が表示されている。
- 操作: 使用素材構成と演出を直す修正依頼を出す。
- 期待結果:
  - 元の動画生成前確認は修正依頼済みになる。
  - 使用素材構成、演出作成、微調整のAI工程が作り直される。
  - 新しい動画生成前確認は新しい微調整工程を参照する。
  - 古い動画生成前確認は承認できない。

### UC-P0-08 生成前確認から既存内容候補へ戻り、別内容で再作成できる

- 前提: 動画生成前確認が表示され、過去に内容候補が作られている。
- 操作: 内容を選び直す修正依頼を出し、別内容を選ぶ。
- 期待結果:
  - 内容候補整理を再実行しない。
  - 既存内容候補を使った新しい内容選択確認が作られる。
  - 別内容を選ぶまで使用素材構成は実行されない。
  - 別内容選択後、新しい構成案に選び直した内容候補IDが反映される。
  - 古い動画生成前確認は承認できない。

### UC-P0-09 失敗したAI工程を再実行できる

- 前提: 内容候補整理が失敗している。
- 操作: 失敗工程を再実行する。
- 期待結果:
  - 失敗した工程と後続の予約済み工程が作り直し対象になる。
  - 同じ工程の新しいAI作業が作られる。
  - 接続情報がまだ無ければ、新しいAI作業も成功扱いにはならず同じ理由で失敗する。

### UC-P0-10 動画生成成功後に生成動画成果物を保存する

- 前提: 動画生成AI工程が作られている。
- 操作: 動画生成成功を報告する。
- 期待結果:
  - 生成動画のファイル参照が保存される。
  - 生成動画の出力情報が保存される。
  - 生成後レビュータブが参照できる成果物になる。

### UC-P0-11 生成後レビューから使用素材構成と演出を作り直せる

- 前提: 確認用動画が生成済みである。
- 操作: 生成後レビューから使用素材構成と演出の作り直しを依頼する。
- 期待結果:
  - 使用素材構成、演出作成、微調整が新しく作られる。
  - 新しい動画生成前確認が確認待ちになる。

### UC-P0-12 生成後レビューから既存内容候補へ戻り、別内容で再作成できる

- 前提: 確認用動画が生成済みで、既存内容候補がある。
- 操作: 生成後レビューから内容選択へ戻り、別内容を選ぶ。
- 期待結果:
  - 内容候補整理を再実行しない。
  - 既存内容候補を使った新しい内容選択確認が作られる。
  - 別内容選択後、新しい構成案に選び直した内容候補IDが反映される。

### UC-P0-13 内容再選択後の実ランナー動画生成が完了する

- 前提: サンプル入力で動画生成前確認まで到達している。
- 操作: 動画生成前確認から内容選択へ戻り、別内容を選び、その編集案で確認用動画を作る。
- 期待結果:
  - 新しい動画生成工程が、新しい動画生成前確認に紐づく。
  - 実ランナーが確認用動画ファイルを書き出す。
  - 動画生成工程が作成中のまま残らず、生成動画成果物として保存される。

## P1 外部連携、異常系、UI状態復元

- YouTube取得失敗時にSTTへ進まない。
- STTサーバー接続失敗時に内容候補整理へ進まない。
- STT応答が空または壊れている場合に後続工程へ渡さない。
- Gemini APIの不正JSON、空応答、タイムアウトを成功扱いにしない。
- UI指定モデルがGemini API呼び出しへ反映される。
- Remotion失敗時に生成後レビューを作らない。
- ページ再読み込み後も現行レビューだけが表示される。
- 二重クリックでAI工程が二重作成されない。

## P2 実連携、品質評価、性能検証

- 実YouTube URLでの取得。
- 実STTサーバーでの文字起こし精度確認。
- 実Gemini APIでのモデル比較と品質評価。
- Remotionの視覚差分確認。
- 長尺動画と大容量動画の性能確認。
- 複数ブラウザでのUI確認。

## 実行方法

```bash
pnpm run scenario:agent
```

テストは一時runtimeと一時バックエンドを使う。通常の `runtime/state.json` は変更しない。
