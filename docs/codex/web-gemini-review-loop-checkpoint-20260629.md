# Web Geminiレビュー実装済み復帰点 2026-06-29

## この時点の意味

完成動画を見た後に、Web Geminiの演出レビューを保存し、そのレビューを使って演出作成前から作り直せる状態。

まだ実Web Geminiへの動画アップロードは、外部送信の明示許可が必要なため自動実行確認まではしていない。ただし、Edge操作の準備確認、CDP診断、レビュー保存、レビュー反映、失敗ログ、現在動画との一致確認、画面表示、自動テストは入っている。

## できること

- 完成動画画面で、Web Geminiレビューの保存状態を見られる。
- AIエージェントが残したレビュー実行ログを画面で見られる。
- レビュー未取得、準備済み、実行中、失敗、保存済みを区別できる。
- 保存済みレビューの改善指示を編集して、演出作成前から再生成できる。
- 反映済みレビューでは、いつ作った編集コピーへ反映済みかを画面で確認できる。
- Web Geminiへ送る依頼文と外部実行手順を画面ログで確認できる。
- 外部実行手順は対象の下書きID付きで残るため、複数の編集コピーがある時に送信対象を追える。
- レビュー実行ログが壊れている場合は、未取得扱いにせずエラーとして表示する。
- レビュー本文が空の場合は、レビュー本体を保存せず失敗ログだけを残す。
- 古い完成動画に対するレビューや実行ログは、現在の完成動画へ反映できない。

## 確認済み

- `corepack pnpm test` で、Web Geminiレビュースクリプト単体テストと全体シナリオテストが成功する。
- `corepack pnpm run type-check` が成功する。
- `corepack pnpm run web-gemini:review:diagnose` で、外部送信なしにEdge操作、Gemini画面、対象下書きID付き外部実行手順を確認できる。
- ブラウザで完成動画画面を開き、レビュー準備済み表示と動画表示に画面エラーがないことを確認した。
- 標準テストで、古いレビュー本体、古い実行ログ、古いレビューの反映拒否を確認した。
- 標準テストで、Web Geminiレビューの改善指示が再キュー化された演出作成工程の入力に入ることを確認した。

## まだ残っていること

- 実際の完成動画をGoogle Gemini Webへアップロードしてレビュー回答を取得する確認。
- Web Geminiレビューで得た改善指示を、実LLMの演出作成品質へ反映する本番確認。
