# ZEV2 UI再構成仕様

作成日: 2026-06-14
管理主体: Codex

## 目的

現在のUIは、依頼、状態、承認、成果物、JSON、入力欄が1画面に並び、ユーザーが「今どの段階で、何を評価すればよいか」を把握しづらい。

この再構成では、処理の高度化ではなく、ユーザーが工程ごとに評価できる画面へ直す。

## 採用する画面構成

- 画面上部に全体の流れ図を置く。
- 流れ図の下に、工程ごとの表示切り替えを置く。
- 各工程では、主な問いを1つだけ先頭に出す。
- 判断材料、成果物、JSON、内部情報は主な問いの下に置く。
- 完了済み工程にも戻って確認できるようにする。

## 初期実装で扱う工程

| 表示 | 主な問い |
| --- | --- |
| 依頼 | この内容でショート作成を始めるか |
| 内容選択 | どの出来事、話題、反応で切り抜きを作るか |
| 使用素材確認 | AIが見つけた素材で演出作成へ進めてよいか |
| 動画生成前確認 | この構成と演出案で確認用動画を作ってよいか |
| 生成動画確認 | 生成された確認用動画を見て、次に直す点は何か |

## 初期実装でやらないこと

- 実STT
- 実Gemini
- 本格動画品質改善
- D4チャット
- D4-lite
- 自動投稿
- ダッシュボード
- 評価スコアリング
- タイムライン自由編集

## UI文言方針

主表示では、開発者向けの役割名や内部状態名を使わない。

- 「APIへ渡す」ではなく「この内容で作成を始める」。
- 「review_required」ではなく「確認待ち」または「承認すると次へ進む」。
- 「成果物」ではなく「確認材料」。
- 「AIの判断」ではなく「提案」。
- 「工程」ではなく「段階」。

## JSON表示方針

JSONは主表示にしない。補助情報として開閉できるようにする。

- 展開後は同じ位置から閉じられる。
- JSON本文と同時に、主要項目の意味を表示する。
- 項目説明は確認材料の種類に合わせて変える。
- ユーザーが項目名を知らなくても、内容候補、使用素材構成、演出案、生成対象、変更理由を読めるようにする。

## 実装対象

- `client/src/App.vue`
- 必要に応じて `docs/codex/zev2-progress.md`
- 必要に応じて `docs/codex/zev2-decisions.md`

## 確認条件

- 上部の流れ図で現在位置が分かる。
- 下部の切り替えで各段階に戻れる。
- 各段階の先頭に主な問いがある。
- 依頼入力欄が常時表示されない。
- 確認材料は段階ごとに絞られている。
- JSONは開閉でき、項目説明が同時に読める。
