# task-005 成果物保存とFileRef実体管理

作成日: 2026-05-31

## 目的

- AIエージェントが作った成果物を、状態ファイルではなく成果物保存領域へ置けるようにする。
- `FileRef` が単なる文字列ではなく、後工程が安全に参照できる成果物参照になるようにする。
- 大きなJSON、音声、動画、画像を状態へ埋め込まない境界を固定する。

## 作業スコープ

今回やること:

- 成果物保存先のルート、命名、作業IDとの関係を決める。
- `complete` で受け取る成果物参照の検証ルールを追加する。
- 成果物種別ごとに、保存すべきMIME typeと参照形式を整理する。
- 後工程が前工程の成果物参照を取得できるAPIまたは状態形式を決める。
- 保存失敗、参照不正、成果物欠落時のエラーを定義する。
- 状態には成果物本文ではなく参照だけを残すことをテストする。

今回やらないこと:

- クラウドストレージ接続を必須にしない。
- 実STT、実LLM、動画生成を実行しない。
- 成果物ビューアやダウンロードUIはこのタスクでは作らない。
- 秘密情報、認証情報、個人情報を成果物参照へ入れない。

## 完了条件

- 成果物保存方針がドキュメント化されている。
- `FileRef` が後工程の入力として使える情報を持つ。
- 成果物本文が状態ファイルへ埋め込まれないことを確認できる。
- 不正な成果物参照がAPIで拒否される。
- 成果物参照が欠落した作業を後工程が実行しない。
- `pnpm run type-check` と対象テストが成功する。
