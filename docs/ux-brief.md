# zev2 想定ユーザ体験

ユーザーは、動画に対してやってほしいことを短く入力し、AIエージェントへ渡す。

AIエージェントの状況と依頼入力は左右に分け、入力導線を崩さない。

ボタンを押した直後、画面は前回の結果ではなく、今渡した依頼に切り替わる。

処理中は、AIエージェントが何をしている段階かが変化として見える。

完了時は、単に「完了」ではなく、今回の依頼でどこまで終わったのかが分かる。

件数だけを主表示にせず、今どうなっていて次に何をすればいいかを見せる。

同じ意味の見出しを重ねず、主画面は要点だけにする。

成功済みの工程は主表示では圧縮し、必要な時だけ開いて確認できる。

過去の実行はメイン画面に詰め込まず、実行履歴から選んで確認する。

現時点ではdry-runなので、本物の動画生成ではなく、AIエージェントが最後まで処理経路を通したことが分かる体験を目標にする。
