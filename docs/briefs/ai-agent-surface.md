# AIが見える情報とAPI

## AIが見えるもの

- GET /api/agent-requests/next: 次に実行する承認済み作業。
- GET /api/state: 依頼、作業状態、成果物参照、失敗理由。
- GET /api/workflow: 工程の順番。

全体状態は開発確認用。成果物本体、動画本体、STT全文、LLM全文は入れない。

## AIが実行するAPI

- GET /api/agent-requests/next: 作業を探す。
  - 返る内容: ID、工程、工程名、対象動画、依頼内容、状態。
  - 作業がなければ空。
  - 工程: 動画取り込み、STT、候補探索、Gemini候補確認、演出付与、微調整、動画生成。
  - 状態: 作業待ち、前工程待ち、実行中、完了、失敗。
- POST /api/agent-requests/:id/claim: 作業を引き受ける。
  - ID: 作業を探した時に返ったID。
  - 成功後: 実行中になる。
- POST /api/agent-requests/:id/complete: 結果と成果物参照を返す。
  - ID: 引き受けた作業のID。
  - 送る内容: 結果の意味。
  - 成功後: 完了になる。
- POST /api/agent-requests/:id/fail: 失敗理由を返す。
  - ID: 失敗した作業のID。
  - 送る内容: 失敗理由。
  - 成功後: 失敗になる。

## アプリケーションが使うAPI

- POST /api/request-drafts
  - 送る内容: 依頼内容、対象動画、尺、候補数、方針、動画生成まで含めるか。
  - 尺: 60秒以内、45秒以内、30秒以内。
  - 候補数: 3候補、5候補、1候補。
  - 方針: ショート向け、字幕重視、安全確認。
  - 成功後: 下書きが作られる。
- POST /api/request-drafts/:id/approve
  - ID: 作成した下書きのID。
  - 成功後: 承認済みになり、AIエージェント用の作業キューが作られる。

依頼作成と承認は、アプリケーションが人間の操作を受けて実行する。
AIエージェントの実行処理は、承認後の作業だけを取得して実行する。

## 現時点

実行処理は仮実行。実STT、実LLM、実Gemini候補確認、実動画生成はまだ接続しない。
