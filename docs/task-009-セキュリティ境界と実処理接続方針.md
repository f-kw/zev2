# task-009 セキュリティ境界と実処理接続方針

作成日: 2026-05-31
更新日: 2026-06-29

## 目的

- AIエージェントAPIに認証、権限、秘密情報管理の境界を入れる。
- STT、LLM、動画生成へ接続する前に、どの処理がどの権限で何を読めるかを決める。
- 実処理接続が始まっても、backendが秘密情報や巨大成果物を無制限に抱えないようにする。

## 作業スコープ

今回やること:

- 人間UIとAIエージェントAPIで必要な権限を分ける。
- AIエージェント識別子、認証方式、トークン保管場所を決める。
- APIレスポンスに秘密情報を含めないことを確認する。
- STT、LLM、動画生成へ渡してよい入力と渡してはいけない入力を整理する。
- 実処理接続時に必要な環境変数、認証情報、保存先をドキュメント化する。
- 実処理接続の開始条件を、API契約、排他制御、成果物保存、ログが完了していることにする。

今回やらないこと:

- このタスクだけで実STT、実LLM、動画生成を接続しない。
- APIキーやトークンをリポジトリへ保存しない。
- 認証情報をUIへ表示しない。
- 外部公開や本番デプロイは行わない。

## 完了条件

- 人間UI用APIとAIエージェント用APIの権限境界が説明できる。
- AIエージェントAPIに最低限の認証方針がある。
- 秘密情報を状態、ログ、成果物参照へ混ぜないルールがある。
- STT、LLM、動画生成へ接続するための前提条件が整理されている。
- 実処理接続前に完了すべきタスクが `docs/order.md` から追える。
- `pnpm run type-check` と対象テストが成功する。

## 実装結果

- 人間UI用APIとAIエージェント実行APIを分ける。
- AIエージェント実行APIは、`GET /api/agent-requests/next`、`claim`、`complete`、`fail`。
- 人間UI用APIは、状態確認、依頼作成、下書き却下、人間確認、作業中止、再実行、Web Geminiレビュー準備。
- `ZEV2_AGENT_API_TOKEN` をbackendに設定すると、AIエージェント実行APIだけBearerトークンが必要になる。
- runnerにも同じ `ZEV2_AGENT_API_TOKEN` を設定すると、AIエージェント実行APIへ `Authorization: Bearer ...` を送る。
- `ZEV2_AGENT_API_TOKEN` が未設定の場合は、ローカル開発用に認証なしで動く。
- トークン、Gemini APIキー、STT接続情報は、状態、AI操作ログ、成果物参照、作業履歴APIへ保存しない。
- 実処理接続時の秘密情報は環境変数に置き、`config/runtime.jsonc` には書かない。

## 実処理接続で渡してよいもの

- STTへ渡してよいもの: 入力動画から抽出した音声、言語設定、処理対象の一時ファイル。
- Gemini APIへ渡してよいもの: 確認済みの動画断片、使用素材構成案、演出作成に必要な短い指示。
- 動画生成へ渡してよいもの: 編集案、生成用調整結果、入力動画参照、成果物出力先。

## 渡してはいけないもの

- APIキー、トークン、認証情報。
- 状態ファイル全体。
- STT全文やGemini応答全文を、必要以上に別工程へ丸ごと渡すこと。
- UI表示のためだけの一時状態。
- 失敗した工程を成功に見せる代替データ。

## 実処理接続の開始条件

- task-002: AIエージェントAPI契約テストと仕様固定が完了している。
- task-004: claim排他制御と復旧が完了している。
- task-005: 成果物保存とFileRef実体管理が完了している。
- task-006: AI操作ログと監査履歴が完了している。
- task-007: 人間制御APIとUI整理が完了している。
- task-009: AIエージェントAPI認証境界と秘密情報ルールが確認済みである。

## 確認方法

- `pnpm run type-check`
- `pnpm run scenario:agent`
- `pnpm test`

シナリオテストでは、`ZEV2_AGENT_API_TOKEN` 設定時にAIエージェント実行APIが認証なしでは拒否され、人間UI用の状態確認は認証なしで通ること、状態APIへトークンが混ざらないことを確認する。
