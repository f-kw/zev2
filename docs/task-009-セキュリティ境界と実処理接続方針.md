# task-009 セキュリティ境界と実処理接続方針

作成日: 2026-05-31

## 目的

- AIエージェントAPIに認証、権限、秘密情報管理の境界を入れる。
- STT、LLM、動画生成へ接続する前に、どの処理がどの権限で何を読めるかを決める。
- 実処理接続が始まっても、backendが秘密情報や巨大成果物を無制限に抱えないようにする。

## 作業スコープ

今回やること:

- 人間UIとAIエージェントAPIで必要な権限を分ける。
- AIエージェント識別子、認証方式、トークン保管場所を決める。
- APIレスポンスに秘密情報を含めないことを確認する。
- STT、LLM、動画生成へ渡してよい入力と渡してはいけない入力を整理する。
- 実処理接続時に必要な環境変数、認証情報、保存先をドキュメント化する。
- 実処理接続の開始条件を、API契約、排他制御、成果物保存、ログが完了していることにする。

今回やらないこと:

- このタスクだけで実STT、実LLM、動画生成を接続しない。
- APIキーやトークンをリポジトリへ保存しない。
- 認証情報をUIへ表示しない。
- 外部公開や本番デプロイは行わない。

## 完了条件

- 人間UI用APIとAIエージェント用APIの権限境界が説明できる。
- AIエージェントAPIに最低限の認証方針がある。
- 秘密情報を状態、ログ、成果物参照へ混ぜないルールがある。
- STT、LLM、動画生成へ接続するための前提条件が整理されている。
- 実処理接続前に完了すべきタスクが `docs/order.md` から追える。
- `pnpm run type-check` と対象テストが成功する。
