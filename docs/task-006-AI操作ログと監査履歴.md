# task-006 AI操作ログと監査履歴

作成日: 2026-05-31

## 目的

- AIエージェントがどの作業をいつ取得し、どのように完了または失敗したかを追えるようにする。
- 失敗調査、再実行判断、人間確認のために、状態変化の履歴を残す。
- ログに秘密情報や巨大な成果物本文を残さない境界を固定する。

## 作業スコープ

今回やること:

- 作業作成、承認、取得、完了、失敗、復旧のイベントを記録する。
- イベントには時刻、作業ID、作業種別、状態変化、短い説明を保存する。
- `GET /api/state` とは別に、必要ならログ確認APIを追加する。
- 失敗理由と成果物参照IDをログから追えるようにする。
- ログへ保存してよい情報と保存してはいけない情報をドキュメント化する。
- ログが巨大化しない保存方針を決める。

今回やらないこと:

- 外部ログサービスへ送信しない。
- STT全文、LLM全文、動画メタデータ全文、APIキー、トークンを保存しない。
- ログ検索UIはこのタスクでは作らない。
- 実STT、実LLM、動画生成を実行しない。

## 完了条件

- API状態遷移ごとに監査イベントが残る。
- 失敗時に、どの作業がどの理由で止まったか確認できる。
- 成果物参照が作られた場合、どの作業由来か追える。
- ログに秘密情報や巨大本文を保存しないことをテストまたはレビューで確認できる。
- `pnpm run type-check` と対象テストが成功する。
